Office File
Vault Office QuickStart enables the creation and simultaneous editing of files and documents based on templates. The templates are managed in the vault.

Define the storage location
A file is always created in the currently selected directory. New files can be created from the folder tree or in the list view. The following example shows the file creation in the directory "Presentations"
 
Because the storage location and file name are already determined the "filename" field will be read-only.
   
Document Type Selection
The New File command is context dependent (= where in Vault do you create the document) and dependent from the used Vault application: Vault Workgroup / Professional order Vault Office. Only Vault Workgroup and Professional allow the creation and editing of CAD files.
In Vault Office Quickstart is only the document type "Office" available. When using many templates it is recommended to subdivide the office category.
All non-CAD document types are here available (the list is not limited to "Office" documents as long as it does not contain any CAD format).
The Category field will not be displayed as the category will be defined by the combination of Document Type and template. Therefore is an explicit selection of the Category not necessary.

Required Fields
When creating a new file only the Title is required. This behaviour is the same for all document types.
 
The Title field requires a minimum length of 3 characters. After meeting this requirement and exiting the field the OK button will be activated.

Navigation
You can use the usual keys to navigate within and close the dialog box:
 Tab - move from field to field
 Enter for line breaks within the field description (multi line text)
 F1 to access the help files
 Enter key close and save the changes to the file
 Esc to cancel and close the dialog box
 
Numbering
Files are numbered sequentially. This serves on one hand to ensure unique file names, on the other hand, the file number can also be used part number for parts, components or drawings.
  
Numbering schemes can be defined as desired. More information for Administrators can be found here: Administration - File-Numbering.
Edit File Datasheet
IIn the datasheet all properties displayed. Read-only properties are inactive; For example, CAD features, such as weight or dimensions, which are automatically determined in the CAD system, should not be changed in the data sheet.
Protected files (checked out or released) will display of datasheet, but the OK button will not be active.
