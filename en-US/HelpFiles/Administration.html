Upgrade Quickstart 2014 -> 2015 R1 / R2
The configuration for Quickstart 2014 and 2015/2015 R2 are separated. But for QuickStart 2015 RTM and 2015 R1 or R2 we are using the same configuration location.
Please note that the temporary file "Folder.XML" needs to be deleted from the users Roaming Profile as this is already created by Quick Start 2014 or 2015 RTM. Simply navigate in Windows Explorer to the% temp% directory and search for "Folder.xml". Delete this file. When you first start QuickStart 2015 the new template of this file will be saved for the user and used.

Multi Language
Vault Quickstart uses the multi language functionality from the Vault Data Standard. There are different language versions of Vault QuickStart available.
Vault QuickStart links the language of the interface with the language of the vault. Therefore the language setting for the UI and for the database must always be the same.

ERP Tab (Since 2015 R2)
Since Vault Quickstart 2015 R2 we provide the new ERP tab that shows you an example on how ERP related information can be visualised on a document in Vault, regardless of the category or document type.
In the directory. \ Templates \ Vault \ ERP you will find the guidelines on how to install the tab and use the ERP sample records.
The sample dataset (sample xml file) uses the same XML structure used when importing or exporting item data from Vault. The associated script file  reads from the XML elements. Change this file according to the structure of the available data from ERP. Direct querying to the ERP system is also possible (e.g. an SQL query or API access to the ERP System).

Object Numbering (Since 2015 R2)
Vault Quickstart uses object numbering. Therefore a numbering scheme with the same name as the object needs to be available. For example the "project" category has an active numbering scheme called “project. Disabling this will make a manual entry in the Object number field possible. The same applies for all custom objects, such as tasks, organizations etc.

File Numbering (Since 2015 R2)
In Vault QuickStart sequential numbering for files is used. Name and layout of the scheme can be changed independently. However it is crucial that the schema for file numbering is enabled by default .
All other Number schemes server for the Object Numbering (Projects, tasks etc.) or are used to set different numbering schemes according the document type . Follow the instructions in the Vault Help system.

File Templates
Vault Quickstart is configured for the creation of Inventor, AutoCAD Mechanical und Microsoft Office documents starting from pre-configured templates.
  
This template structure is shown in the file creation dialog boxes: File.XAML and FileOffice.XAML. An addition or expansion of the template structure or document types must be configured in either the XAML file as the Vault template folder structure.
 
Sequence of the selection list
The order of the templates corresponds to the order in which the originals were added to this directory if no sorting is activated. For example, if an extra template file is added to Inventor templates, this will be listed at the bottom of the list.
